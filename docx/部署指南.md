# 宠物领养系统 — 部署指南

## 项目结构

```
pet_adoption_system/
├── server/              # Spring Boot 后端（Java 17+，端口 8080）
├── client/              # Vue3 + Element Plus 管理后台（Vite，端口 5173）
├── uniapp/              # uni-app 移动端
├── docx/                # 文档 & 初始化 SQL
├── nginx.conf           # Nginx 配置（前后端合并部署用）
├── entrypoint.sh        # 容器启动脚本（前后端合并部署用）
└── build-deploy.sh      # 一键构建部署包脚本
```

## 一、本地开发

### 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 17+ |
| Maven | 3.9+ |
| Node.js | 20.19+ |
| MySQL | 8.0+ |
| Nginx | 1.22+ |

#### 安装依赖（Debian 12）

```bash
# JDK 17
sudo apt update
sudo apt install openjdk-17-jdk -y

# Maven
sudo apt install maven -y

# Node.js 20+（从 NodeSource 安装）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install nodejs -y

# Nginx
sudo apt install nginx -y

# 验证
java --version
mvn --version
node --version
npm --version
nginx -v
```

#### 安装依赖（Windows）

1. JDK 17：[Adoptium Temurin](https://adoptium.net/download/) 下载安装，勾选 "Add to PATH"
2. Maven：[官网下载](https://maven.apache.org/download.cgi)，解压后将 `bin/` 加入 PATH
3. Node.js：[官网下载](https://nodejs.org/) LTS 版本安装

### 1.1 初始化数据库

在本地 MySQL 中执行 `docx/init.sql`。

默认管理员：`admin` / `123456`

### 1.2 配置数据库连接

编辑 `server/src/main/resources/application-dev.properties`，改为你的本地 MySQL 连接：

```properties
spring.datasource.url=jdbc:mysql://你的IP:3306/pet_adoption_db?...
spring.datasource.username=你的用户名
spring.datasource.password=你的密码
```

### 1.3 启动后端

```bash
cd server
mvn spring-boot:run
```

后端启动在 `http://localhost:8080`，默认激活 `dev` profile。

### 1.4 启动前端

```bash
cd client
npm install

# 连本地后端（默认）
npm run dev                 # http://localhost:5173

# 连生产后端
npm run dev:prod            # Vite 代理转发，无跨域
```

Vite 自动将 `/api` 和 `/uploads` 代理到对应后端（见 `vite.config.js`），开发时无跨域问题。

### 1.5 三种运行模式

| 命令 | `.env` 文件 | `VITE_API_BASE` | 后端地址 | 适用场景 |
|------|------------|-----------------|---------|---------|
| `npm run dev` | `.env.development` | 空（走 Vite 代理） | `localhost:8080` | 本地开发 |
| `npm run dev:prod` | `.env.production-local` | 空（走 Vite 代理） | `bltiltwlzlzz.sealosbja.site` | 本地调试生产接口 |
| `npm run build` | `.env.production` | 空（相对路径） | 同域名 Nginx 转发 | 生产构建 |

## 二、生产部署（前后端合并，同域名）

### 2.1 架构

```
浏览器 ──→ :80 (Nginx)
              │
              ├── /            → /app/dist（前端静态文件，SPA fallback）
              ├── /api/*       → localhost:8080（Spring Boot 后端）
              └── /uploads/*   → localhost:8080（上传图片）
```

所有请求同一个域名，Nginx 分流，无跨域问题。

### 2.2 关键文件

| 文件 | 位置 | 作用 |
|------|------|------|
| [nginx.conf](nginx.conf) | 项目根目录 | 托管静态文件 + 反向代理 API |
| [entrypoint.sh](entrypoint.sh) | 项目根目录 | 先启动后端，健康检查通过后启动 Nginx |
| [.env.production](client/.env.production) | client/ | `VITE_API_BASE=` 空，前端走相对路径 |

### 2.3 一键构建

```bash
bash build-deploy.sh
```

脚本自动完成：编译前端 → 打包后端 → 整理为 `deploy/` 目录。

构建后的目录结构（容器文件系统镜像）：

```
deploy/
├── entrypoint.sh                           # 容器启动脚本
├── app/
│   ├── dist/                               # 前端静态文件
│   └── server.jar                          # 后端 jar
└── etc/
    └── nginx/
        └── conf.d/
            └── default.conf                # Nginx 配置
```

### 2.4 手动构建（可选）

```bash
# 1. 构建前端
cd client && npm install && npm run build   # 产物：dist/

# 2. 构建后端
cd ../server && mvn clean package -DskipTests  # 产物：target/server-0.0.1-SNAPSHOT.jar
```

手动拷贝到 `deploy/` 对应目录即可。

### 2.5 部署到 Sealos / 服务器

将 `deploy/` 目录下所有文件上传到容器的 **`/` 根目录**。

容器要求：

- **基础镜像**：需同时包含 Nginx 和 JDK 17（如 `nginx:stable` 安装 `openjdk-17-jre-headless`）
- **端口**：暴露 80
- **启动命令**：`/entrypoint.sh`（已内置 `--spring.profiles.active=prod`）
- **持久化存储**：为 `/app/uploads` 目录挂载持久卷（否则重新部署后图片丢失）

> 在 Sealos 中：应用设置 → 存储 → 添加挂载卷，容器内路径填 `/app/uploads`，容量 1Gi 即可。

### 2.6 数据库（生产）

`application-prod.properties` 中的连接信息：

| 项目 | 值 |
|------|-----|
| Host | `dbconn.sealosbja.site` |
| Port | `45629` |
| User | `root` |
| Database | `pet_adoption_db` |

## 三、Uniapp 移动端

### 3.1 切换环境

编辑 [uniapp/uniapp/utils/config.js](uniapp/uniapp/utils/config.js)：

```js
const ENV = 'dev'    // 开发：http://localhost:8080
const ENV = 'prod'   // 生产：https://bltiltwlzlzz.sealosbja.site
```

> 使用合并部署后，应将 `prod` 地址改为新的统一域名（如 `https://bltiltwlzlzz.sealosbja.site`），因为后端 API 也通过该域名的 `/api` 路径访问。

### 3.2 开发

1. 用 HBuilderX 打开 `uniapp/uniapp/` 目录
2. 运行 → 微信开发者工具

### 3.3 发布

1. HBuilderX → 发行 → 小程序-微信
2. 微信开发者工具 → 上传代码

## 五、GitHub 版本管理

> 仓库地址：`https://github.com/lliuxiangke0210/pet_adoption_system`

### 5.1 首次配置

```bash
cd pet_adoption_system

# 配置 Git 用户信息（首次使用 Git 时需要）
git config user.name "你的用户名"
git config user.email "你的邮箱"

# 如果还没有克隆仓库，先克隆
git clone https://github.com/lliuxiangke0210/pet_adoption_system.git
```

### 5.2 推送本地更新

```bash
cd pet_adoption_system

# 1. 查看当前修改状态
git status

# 2. 将修改的文件添加到暂存区
git add .                    # 添加所有修改
# 或指定文件
git add docx/部署指南.md       # 添加单个文件

# 3. 提交（附带说明信息）
git commit -m "更新部署文档"

# 4. 推送到 GitHub
git push origin main
```

### 5.3 拉取远程更新

```bash
git pull origin main
```
